# Contributing to SCA Module Catalogue

Thank you for your interest in contributing to **SCA Module Catalogue**!
We welcome contributions of all kinds—code, documentation, tests, design, and feedback.

Please take a few minutes to read this guide. Following it helps us maintain a productive and respectful environment.

---

## Quick Start

1. **Fork** this repository
2. **Create a feature branch**: `git checkout -b your-feature-name`
3. **Make your changes**
4. **Commit** clearly: `git commit -m "feat: describe your change"`
5. **Push** to your fork: `git push origin your-feature-name`
6. **Open a Pull Request** with a clear description

---

## What Can I Contribute?

* ✅ Bug fixes
* ✅ New features (open an issue first for major changes)
* ✅ Accessibility improvements
* ✅ Documentation (README, examples, tutorials)
* ✅ Tests
* ✅ Code refactoring or performance improvements
* ✅ Ideas, suggestions, and design feedback

---

## Code Guidelines

* Follow existing code style and naming conventions
* Use clear, meaningful commit messages (e.g., `fix: prevent crash when input is null`)
* Keep pull requests small and focused

---

## Documentation

Good documentation is just as important as code. If you add a new feature, update the relevant documentation:

* Inline comments and function-level docs
* README or other markdown guides
* Example usage if applicable

---


## Communication

* Be respectful and inclusive (see our [Code of Conduct](./CODE_OF_CONDUCT.md))
* Use GitLab Issues to discuss ideas before contributing with bug fixes or change requests. Use the following [template](./ISSUES_TEMPLATE.md) for reporting bugs
* Join discussions in merge requests and issues—feedback is welcome!
